# Review Content Skill

## Purpose

Improve content quality through systematic review, editing, and feedback.

## When to Use

- After first draft complete
- Before publishing
- When updating content
- For quality audits

## Review Types

### Self-Review

**When:** After completing draft

**Focus:**
- Content completeness
- Clarity and flow
- Grammar and spelling
- Technical accuracy

**Process:**
1. Take break (fresh eyes)
2. Read aloud
3. Check against outline
4. Verify all claims

### Technical Review

**When:** For technical content

**Focus:**
- Technical accuracy
- Code examples work
- Commands tested
- Versions current

**Reviewer:** Subject matter expert

### Editorial Review

**When:** Before publishing

**Focus:**
- Style guide compliance
- Grammar and spelling
- Clarity and flow
- Consistency

**Reviewer:** Editor or peer

### Accessibility Review

**When:** Before publishing

**Focus:**
- Alt text present
- Heading hierarchy
- Color contrast
- Screen reader friendly

**Reviewer:** Accessibility specialist

## Review Process

### 1. First Pass - Content

**Check:**
- All topics covered
- Logical flow
- No gaps
- Examples included

**Questions:**
- Does this achieve its goal?
- Is anything missing?
- Is anything unnecessary?

### 2. Second Pass - Structure

**Check:**
- Heading hierarchy
- Paragraph structure
- Section balance
- Transitions

**Questions:**
- Is the organization logical?
- Do sections flow well?
- Are headings descriptive?

### 3. Third Pass - Clarity

**Check:**
- Sentence clarity
- Word choice
- Jargon explained
- Ambiguity removed

**Questions:**
- Is this easy to understand?
- Are terms defined?
- Is the meaning clear?

### 4. Fourth Pass - Mechanics

**Check:**
- Grammar
- Spelling
- Punctuation
- Formatting

**Questions:**
- Are there errors?
- Is formatting consistent?
- Do links work?

## Editing Techniques

### Substantive Editing

**What:** Major content changes

**Actions:**
- Reorganize sections
- Add/remove content
- Rewrite unclear passages
- Improve examples

### Copy Editing

**What:** Sentence-level improvements

**Actions:**
- Fix grammar
- Improve word choice
- Ensure consistency
- Correct punctuation

### Proofreading

**What:** Final error check

**Actions:**
- Catch typos
- Verify formatting
- Check page breaks
- Final polish

## Feedback Delivery

### Comment Types

**Praise:**
```
Great explanation of the complex concept!
This example really clarifies the point.
```

**Suggestions:**
```
Consider adding an example here.
This sentence could be clearer.
```

**Questions:**
```
Is this still current in version 3.x?
Should we link to the related tutorial?
```

**Corrections:**
```
This command is missing the --flag option.
The version number should be 3.2, not 3.1.
```

### Tone Guidelines

**Do:**
- Be specific
- Be constructive
- Explain why
- Suggest solutions

**Don't:**
- Be vague ("fix this")
- Be harsh ("this is wrong")
- Just criticize
- Make it personal

## Review Checklists

### Content Checklist

- [ ] All outline points covered
- [ ] Introduction sets up content
- [ ] Conclusion summarizes key points
- [ ] Examples support concepts
- [ ] Claims are supported
- [ ] No logical gaps
- [ ] Appropriate depth

### Structure Checklist

- [ ] Headings are descriptive
- [ ] Heading hierarchy correct
- [ ] Paragraphs focused
- [ ] Transitions smooth
- [ ] Sections balanced
- [ ] Visual placement logical

### Clarity Checklist

- [ ] Topic sentences clear
- [ ] Jargon explained
- [ ] Sentences readable
- [ ] No ambiguity
- [ ] Active voice used
- [ ] Consistent terminology

### Mechanics Checklist

- [ ] No spelling errors
- [ ] Grammar correct
- [ ] Punctuation correct
- [ ] Formatting consistent
- [ ] Links work
- [ ] Images display

### Technical Checklist

- [ ] Code examples run
- [ ] Commands tested
- [ ] Versions current
- [ ] Links to docs work
- [ ] No deprecated features
- [ ] Security considerations noted

## Common Issues

### Content Issues

**Problem:** Gaps in explanation
**Fix:** Add missing steps or context

**Problem:** Tangents
**Fix:** Remove or move to separate article

**Problem:** Too shallow
**Fix:** Add depth, examples, details

### Writing Issues

**Problem:** Long sentences
**Fix:** Split into shorter sentences

**Problem:** Passive voice
**Fix:** Rewrite in active voice

**Problem:** Unclear antecedents
**Fix:** Replace pronouns with nouns

### Technical Issues

**Problem:** Outdated examples
**Fix:** Update to current versions

**Problem:** Untested code
**Fix:** Run and verify all examples

**Problem:** Broken links
**Fix:** Update or remove links
