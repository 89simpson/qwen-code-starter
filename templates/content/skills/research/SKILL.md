# Research Skill (Content)

## Purpose

Conduct thorough research to gather accurate, comprehensive information for content creation.

## When to Use

- Starting new content pieces
- Updating existing content
- Verifying technical claims
- Gathering examples and case studies
- Competitive content analysis

## Research Process

### 1. Define Scope

**Questions to Answer:**
- What is the topic?
- Who is the audience?
- What do they need to know?
- What is the content format?
- What is the deadline?

**Research Plan:**
```markdown
## Research Plan: [Topic]

### Objective
[What we need to learn]

### Key Questions
1. [Question 1]
2. [Question 2]

### Sources to Check
- [Source type 1]
- [Source type 2]

### Time Box
[Time limit for research]
```

### 2. Gather Sources

**Primary Sources:**
- Official documentation
- API specifications
- Original research
- Direct interviews

**Secondary Sources:**
- Technical blogs
- Tutorial articles
- Conference talks
- Books

**Tertiary Sources:**
- Wikipedia (for discovery)
- Aggregator sites
- Summary articles

### 3. Evaluate Sources

**Credibility Check:**
- Author qualifications
- Publication reputation
- Publication date
- Citations provided

**Accuracy Check:**
- Cross-reference claims
- Verify technical details
- Test code examples
- Check for updates

### 4. Organize Findings

**Source Document:**
```markdown
# Research: [Topic]

## Key Findings
1. [Finding 1] - [Source]
2. [Finding 2] - [Source]

## Sources

### Primary
1. [Title](URL)
   - Key points: [Summary]
   - Verified: [Yes/No]

### Secondary
1. [Title](URL)
   - Key points: [Summary]
   - Notes: [Any concerns]

## Quotes
> "Relevant quote" - Author

## Examples
- Code example from [source]
- Case study from [source]

## Gaps
- [Information still needed]
```

## Research Techniques

### Documentation Analysis

**Reading Strategy:**
1. Skim table of contents
2. Read introduction/overview
3. Focus on relevant sections
4. Note important details
5. Check for updates/version

**Note-Taking:**
```markdown
## Source: [Documentation Name]

### Relevant Sections
- Section 1: [Key point]
- Section 2: [Key point]

### Code Examples
```language
# Example to reference
```

### Gotchas
- [Important caveat]
- [Version requirement]
```

### Competitive Analysis

**What to Analyze:**
- Content structure
- Depth of coverage
- Examples provided
- User engagement
- Gaps in coverage

**Analysis Template:**
```markdown
## Competitive Analysis: [Topic]

### Competitor 1
- URL: [link]
- Strengths: [list]
- Weaknesses: [list]
- Coverage: [topics covered]

### Opportunities
- [What they miss that we can cover]
- [How we can do better]
```

### Expert Interviews

**Preparation:**
- Prepare questions in advance
- Research interviewee's work
- Test recording equipment
- Get permission to quote

**Question Types:**
- Background/context
- Technical details
- Best practices
- Common mistakes
- Future trends

**Follow-up:**
- Send transcript for review
- Get quote approval
- Verify technical claims
- Thank them publicly

## Quality Standards

### Source Quality
- Prefer primary sources
- Verify with multiple sources
- Check publication dates
- Consider source credibility

### Coverage Quality
- All key topics covered
- Multiple perspectives
- Current information
- Practical examples

### Documentation Quality
- Complete citations
- Clear organization
- Easy to reference
- Gaps identified

## Time Management

### Time Allocation

| Activity | Percentage |
|----------|------------|
| Source discovery | 30% |
| Source evaluation | 20% |
| Note-taking | 30% |
| Organization | 20% |

### Time-Boxing

**Short Research (1-2 hours):**
- Focused topic
- Limited sources
- Quick turnaround

**Standard Research (4-8 hours):**
- Comprehensive topic
- Multiple source types
- Standard content piece

**Deep Research (2+ days):**
- Complex topic
- Expert interviews
- Major content piece

## Common Pitfalls

### Analysis Paralysis
- Too many sources
- Endless research
- No content creation

**Solution:** Time-box, set source limits

### Source Bias
- Only one perspective
- No contradictory info
- Confirmation bias

**Solution:** Seek opposing views

### Outdated Information
- Old documentation
- Superseded practices
- Version mismatches

**Solution:** Check dates, verify current
