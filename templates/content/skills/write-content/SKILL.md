# Write Content Skill

## Purpose

Transform outlines and research into clear, engaging, well-written content.

## When to Use

- After outline approval
- For first drafts
- For content revisions
- For updating existing content

## Writing Process

### 1. Preparation

**Before Writing:**
- Review outline thoroughly
- Have research handy
- Eliminate distractions
- Set writing goals (time or word count)

**Environment:**
- Quiet space
- Necessary tools ready
- Reference materials accessible

### 2. Drafting

**First Pass:**
- Write without editing
- Follow the outline
- Get ideas down
- Don't worry about perfection

**Tips:**
- Use placeholder [TODO] for uncertain parts
- Mark sections needing research
- Keep momentum going

### 3. Revision

**Self-Edit Pass:**
- Fill in TODOs
- Complete research
- Improve clarity
- Tighten prose

**Polish Pass:**
- Check grammar/spelling
- Verify consistency
- Improve transitions
- Final read-through

## Writing Guidelines

### Clarity

**Sentence Structure:**
- Keep sentences focused (one idea each)
- Vary sentence length
- Use active voice
- Avoid nominalizations

**Before:**
```
The implementation of the feature was done by the team.
```

**After:**
```
The team implemented the feature.
```

### Conciseness

**Remove Filler:**
- "In order to" → "To"
- "Due to the fact that" → "Because"
- "At this point in time" → "Now"
- "In the event that" → "If"

**Before:**
```
It is important to note that in most cases, 
users will need to complete the process in order to proceed.
```

**After:**
```
Users must complete the process to proceed.
```

### Engagement

**Opening Hooks:**
- Surprising fact
- Relatable problem
- Question
- Brief story

**Maintain Interest:**
- Vary paragraph length
- Use examples
- Add visuals
- Include subheadings

## Content Types

### Explanatory Writing

**Purpose:** Help reader understand

**Techniques:**
- Start with familiar concepts
- Build to new ideas
- Use analogies
- Provide examples

**Structure:**
```
1. What it is
2. Why it matters
3. How it works
4. Example
5. When to use
```

### Instructional Writing

**Purpose:** Help reader do something

**Techniques:**
- Imperative mood
- Step-by-step format
- Clear expectations
- Troubleshooting help

**Structure:**
```
1. Goal
2. Prerequisites
3. Steps (numbered)
4. Verification
5. Next steps
```

### Persuasive Writing

**Purpose:** Convince reader

**Techniques:**
- State position clearly
- Provide evidence
- Address counterarguments
- Call to action

**Structure:**
```
1. Problem/opportunity
2. Proposed solution
3. Benefits
4. Evidence
5. Address concerns
6. Call to action
```

## Section Writing

### Introductions

**Goals:**
- Capture attention
- Provide context
- State purpose
- Preview content

**Template:**
```
[Hook - surprising fact, question, or problem]

[Context - why this matters]

[Thesis - what this article covers]

[Preview - brief outline of sections]
```

### Body Paragraphs

**Structure:**
```
Topic sentence (main point)
↓
Supporting details
↓
Example or evidence
↓
Transition to next point
```

### Conclusions

**Goals:**
- Summarize key points
- Reinforce main message
- Provide next steps
- Leave lasting impression

**Template:**
```
[Restate main point in new words]

[Summarize key takeaways]

[Call to action or next steps]

[Final thought or forward look]
```

## Code and Technical Content

### Code Examples

**Best Practices:**
- Complete, runnable code
- Syntax highlighting
- Comments for complex parts
- Show expected output

**Format:**
````markdown
```python
def example():
    """Clear example function."""
    return "result"
```

This function returns a string result.
````

### Technical Explanations

**Approach:**
1. High-level concept first
2. Then details
3. Concrete example
4. Edge cases

**Avoid:**
- Jargon without explanation
- Assumed knowledge
- Vague descriptions

## Writing Challenges

### Writer's Block

**Solutions:**
- Skip to easier section
- Write out of order
- Set timer for 10 minutes
- Talk through the point

### Unclear Concepts

**Solutions:**
- Draw a diagram
- Create an analogy
- Explain to imaginary person
- Find better example

### Too Much Information

**Solutions:**
- Split into multiple articles
- Move details to appendix
- Link to reference docs
- Focus on common cases

## Quality Checklist

### Before Submitting

**Content:**
- [ ] All outline points covered
- [ ] Examples included
- [ ] Claims supported
- [ ] No gaps in logic

**Writing:**
- [ ] Clear topic sentences
- [ ] Smooth transitions
- [ ] Consistent tone
- [ ] No repetition

**Technical:**
- [ ] Code examples work
- [ ] Links functional
- [ ] Facts verified
- [ ] Versions current

**Mechanics:**
- [ ] Grammar correct
- [ ] Spelling correct
- [ ] Punctuation correct
- [ ] Formatting consistent

## Productivity Tips

### Writing Sprints

**Technique:**
1. Set timer (25 min)
2. Write without stopping
3. Take break (5 min)
4. Repeat

**Goal:** Momentum over perfection

### Reverse Outline

**After Drafting:**
1. Extract outline from draft
2. Compare to original outline
3. Identify gaps or tangents
4. Revise accordingly

### Read Aloud

**Benefits:**
- Catches awkward phrasing
- Finds run-on sentences
- Identifies repetition
- Tests flow
