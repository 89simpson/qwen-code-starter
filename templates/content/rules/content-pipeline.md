# Content Pipeline Rule

## Purpose

Define a consistent workflow for content creation from ideation to publication and maintenance.

## Pipeline Stages

### 1. Research

**Objective:** Gather accurate, comprehensive information

**Activities:**
- Identify topic scope
- Collect primary sources
- Review existing content
- Interview subject matter experts
- Gather code examples (if technical)

**Deliverables:**
- Source document with links
- Key points outline
- Technical verification notes
- Competitor content analysis

**Exit Criteria:**
- [ ] All sources collected
- [ ] Technical accuracy verified
- [ ] Scope defined
- [ ] Audience identified

### 2. Outline

**Objective:** Structure content for clarity and flow

**Activities:**
- Define main sections
- Plan subsections
- Identify key takeaways
- Plan examples and illustrations
- Estimate reading time

**Deliverables:**
- Detailed outline with hierarchy
- Section descriptions
- Example placements
- Visual element plan

**Exit Criteria:**
- [ ] Logical flow established
- [ ] All key points included
- [ ] Examples planned
- [ ] Outline reviewed

### 3. Draft

**Objective:** Create complete first version

**Activities:**
- Write all sections
- Add code examples
- Create diagrams/illustrations
- Insert placeholder links
- Add metadata

**Deliverables:**
- Complete draft
- All examples included
- Images/diagrams added
- Metadata complete

**Exit Criteria:**
- [ ] All sections written
- [ ] Examples functional
- [ ] Images included
- [ ] No TODOs remaining

### 4. Review

**Objective:** Improve quality and accuracy

**Activities:**
- Self-edit for clarity
- Technical review
- Style guide compliance
- Accessibility check
- SEO optimization

**Deliverables:**
- Edited draft
- Review comments addressed
- Style compliance verified
- Accessibility verified

**Exit Criteria:**
- [ ] All reviews complete
- [ ] Comments addressed
- [ ] Style guide followed
- [ ] Accessibility checked

### 5. Publish

**Objective:** Deploy content to audience

**Activities:**
- Final proofread
- Generate build (if static)
- Deploy to staging
- Staging verification
- Deploy to production
- Production verification

**Deliverables:**
- Published content
- Verification report
- Analytics configured
- Social sharing ready

**Exit Criteria:**
- [ ] All checks passed
- [ ] Deployed successfully
- [ ] Links verified
- [ ] Analytics working

### 6. Maintain

**Objective:** Keep content accurate and relevant

**Activities:**
- Monitor feedback
- Track analytics
- Update for accuracy
- Fix broken links
- Refresh examples

**Deliverables:**
- Update log
- Version history
- Feedback responses
- Analytics reports

**Exit Criteria:**
- [ ] Regular audits done
- [ ] Updates applied
- [ ] Links fixed
- [ ] Feedback addressed

## Workflow Management

### Status Tracking

| Status | Description | Next Action |
|--------|-------------|-------------|
| Backlog | Idea identified | Prioritize |
| Research | Gathering info | Complete research |
| Outline | Structuring | Approve outline |
| Draft | Writing | Complete draft |
| Review | Editing | Address feedback |
| Ready | Approved | Schedule publish |
| Published | Live | Monitor |
| Archived | Outdated | Update or remove |

### Priority Levels

**P0 - Critical:**
- Security updates
- Breaking change documentation
- Compliance requirements

**P1 - High:**
- New feature documentation
- Major bug fixes
- User-requested content

**P2 - Medium:**
- Improvements to existing content
- Additional examples
- SEO optimization

**P3 - Low:**
- Nice-to-have content
- Minor clarifications
- Style improvements

## Quality Gates

### Gate 1: Research Complete
- Sources verified
- Technical accuracy confirmed
- Scope appropriate

### Gate 2: Outline Approved
- Logical structure
- Complete coverage
- Audience appropriate

### Gate 3: Draft Complete
- All sections written
- Examples included
- No gaps

### Gate 4: Review Passed
- All feedback addressed
- Style compliant
- Accessibility verified

### Gate 5: Ready to Publish
- Final approval
- All checks passed
- Publishing scheduled

## Collaboration

### Roles and Responsibilities

**Author:**
- Research and write content
- Address feedback
- Maintain after publishing

**Technical Reviewer:**
- Verify technical accuracy
- Validate examples
- Check completeness

**Editor:**
- Style and grammar
- Clarity and flow
- Consistency

**Publisher:**
- Final verification
- Deployment
- Post-publish monitoring

### Handoff Process

```
Research → Author creates outline
Outline → Reviewer approves
Draft → Editor reviews
Review → Author addresses feedback
Ready → Publisher deploys
Published → Author maintains
```

- Technical accuracy rate
- Style compliance score

### Impact Metrics
- Page views
